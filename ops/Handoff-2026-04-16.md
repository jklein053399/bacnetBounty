## If something goes wrong during soak

### "Process died, ports bound, can't restart"
```bash
taskkill //F //IM python.exe
# Wait 30 seconds for sockets to fully release
sleep 30
# Verify ports are free:
netstat -ano -p udp | grep ":47808"
# Should return nothing. If anything still bound, give it another 30s.
# Then restart:
cd /c/Programs/Bacnet-Bounty
nohup py -m simulator > logs/simulator.log 2>&1 &
```

### "Process is alive but stuck (no recent tick heartbeat)"
```bash
# Capture the hung state for debugging:
tail -100 logs/simulator.log > logs/hung_state.log
# Then taskkill and restart as above.
```

### "Memory climbed past 200 MB"
This wasn't expected. Either BAC0 has a per-tick leak we didn't catch, or one of the device `update()` paths is retaining state. Capture `tasklist`, `tail -500 logs/simulator.log`, and flag me. Don't delay ship on this — just send me the data and restart.

### "Log file not rotating after midnight"
Unlikely but possible if system clock is off. Check current time vs log's last mtime. Worst case: delete `simulator.log.*` older than 7 days manually.
