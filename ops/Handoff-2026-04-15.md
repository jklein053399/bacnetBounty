## Session B scope (2026-04-16, Thu)

### Phase 4 — AHU device class (3 instances)

Reference spec: doc 02 §5.

1. **`simulator/ahu_physics.py`** — pure-function module. Inputs: `SiteState`, AHU config (position="single_zone"|"vav", design_cfm, fan_nominal_kw, cooling_tons). Outputs: dataclass with SAT, RAT, MAT, SA pressure, SA airflow, filter ΔP, AHU real power (fan + aux, per doc 03 Flag 1 which redefined this point), fan status, SAT setpoint, SA pressure setpoint, damper positions (OA, cooling, heating), fan VFD command, DX stages.
   - Fan kW: `nominal_kw × (vfd_speed/100)^2.5` per doc 02 §5
   - Use `reference/ahu_model.py` and `reference/generic_ahu.py` as pattern input, not as drop-in
2. **`simulator/devices/ahu.py`** — thin BACnet wrapper. ~18 points per doc 02 §5 (10 AIs/AVs, 2 BIs, 1 BO, 4 AOs roughly). Vendor name "Metro Controls", model "AHU-Sim", firmware "1.0".
3. **Manifest entries in `__main__.py`**: AHU_1 (`200001`, offset 5, position="single_zone", design_cfm=1600, fan_kw=2.5, tons=12), AHU_2 (`200002`, offset 6, position="vav", design_cfm=5600, fan_kw=8.0, tons=40), AHU_3 (`200003`, offset 7, same as AHU_2).
4. **`site_config.json`**: add AHU config section with per-AHU rows keyed by name (per architectural invariant #3 — config-driven uniqueness).
5. **`scripts/verify_ahu.py`** — binds 3 AHUs, reads all ~18 points each, validates: SAT tracks setpoint within ±5°F, fan kW within 0.5–design_kw, fan_status BI tracks fan_command with ~3-sec delay simulated, SA pressure in range.

### Phase 5 — VAV device class (20 instances)

Reference spec: doc 02 §6.

1. **`simulator/vav_physics.py`** — pure-function module. Inputs: `SiteState`, VAV config (zone_position="perimeter"|"interior", design_cfm=500, min_cfm=150, reheat_mbh=15). Outputs: zone temp, airflow, discharge air temp (= parent AHU SAT + reheat delta when valve open), cooling SP, heating SP, airflow SP, damper position, reheat valve position, occupancy status.
2. **`simulator/devices/vav.py`** — thin BACnet wrapper. ~8 points per doc 02 §6.
3. **Manifest entries**: `300001`–`300010` fed from AHU_2 at offsets 8–17; `300011`–`300020` fed from AHU_3 at offsets 18–27. Per-instance phase-shift on zone temp drift so they don't all move in lockstep.
4. **`site_config.json`**: 20 VAV config rows (position tags, offsets) per architectural invariant. Jake explicitly signed off on the long row count — ugly but functional.
5. **`scripts/verify_vav.py`** — spot-check several instances: damper tracks airflow SP, reheat valve active in heating mode, discharge air temp = SAT + reheat delta when reheating, occupancy BI tracks schedule.
